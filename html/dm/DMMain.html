Introduction

The Data Set Manager is a comprehensive software tool designed for the manipulation, archival, and processing of various geographically referenced data.

The Data Set Manager defines a binary, geo-referenced data set format. External data sets must be converted to this format in order to be used by the Data Set Manager. The Data Set Manager includes routines to handle this conversion.

The basic functionality is subdivided into four groups. The input and output routines allow conversion between various external data sources, and the compact, geo-referenced format used by the Data Set Manager. The Utitility routines provide basic file manipulation and miscellaneous background tasks. Operation routines are procedures that are used interactively. These routines include visualization, documentation, and basic analysis capabilities. Finally, the Processes are routines which require a large amount of execution time. Such routines include interpolation and modelling. All Process routines
are executed in the background.

1.0 Basic Usage

The Upper list in the main window shows the data sets which are present in the current directory. These can be selected using the left mouse button, and then the requested operation may be selected from the menu bar. The lower window shows any background data set manager processes that are currently operating.

    • The Area Weighted Average routine is used to interpolate matrix cell data.  Each cell in the interpolated data set will be a combination of the partial areas of the individual cells of the original data set which overlap its area.

    Interpolate List: This list is not editable.  It shows the data sets which are to be interpolated, and the name of the interpolated version.

    Boundaries and Cell Size: The North, South, East, and West entry boxes are used to set the bounds of the region to interpolate.  The 'Dy' and 'Dx' entrys allow the user to specify the latitudinal and longitudinal cell size, respectively.
     

    Cell Reference: The 'Cell Reference' toggles allow the user to indicate whether the supplied boundaries reference the center or the lower left hand cornter of the data cells.

    Missing Values Toggle:  Any missing data values in the original data set must not be interpreted as data.  The 'Missing Values' toggle causes the area weighted average routine to explicitly exclude such points from the average.

    Minimum Valid Cell Area for Inclusion:  If missing values are present in the original data set, certain interpolated cells will not have valid data within their entire area.  This entry controls how much of the
    cell's area must contain data in order for it to be included in the output.

    Maximum Boundaries Button: The 'Maximum Boundaries' button creates the largest possible boundaries given the user specified cell size and boundaries of the original data set.  If multiple data sets have been
    selected, only the first data set's boundary will be used for this calculation.

    Process Option: Selecting 'Process Sequentially' causes the interpolations to be performed one after another.  Selecting 'Process in Parallel' causes all interpolations to be processed simultaneously in the background.  Sequential processing lessens the load on the computer.

    Error Log File: This text field allows the user to specify a name for the error log file.  Since the interpolations are performed in the background, they cannot use windows or text to inform the user of an error condition.  Instead, they write the errors to the specified log file.

    The Bilinear Interpolation routines gives the user the ability to resize the cells of a matrix data set.  Each interpolated point is based on linear distance weightings of the four nearest data points in the original data set. The bilinear algorithm has no extrapolation capabilites, and therefore the region to interpolate must be completely contained by the original data set.

    Interpolate List:  This list is not editable.  It shows the data sets which are to be interpolated, and the name of the interpolated version.

    Boundaries:  The North, South, East, and West entry boxes are used to specify the boundaries for the interpolation.  These boundary values are set to the boundaries of the last selected data set upon entry into the routine.

    Cell Size:  The 'Dy' and 'Dx' fields enable the user to specify the latitudinal and longitudinal cell sizes for the
    interpolation.

    Cell Reference: The 'Cell Reference' toggles indicate whether the boundaries specified by the user are referencing the center or lower left corner of the cells in the data set.  If the 'Lower Left Corner' toggle is set then the boundaries are all increased by a half of a cell to convert them to center referenced coordinates.

    Process Option: Selecting 'Process Sequentially' causes the interpolations to be performed one after another.  Selecting 'Process in Parallel' causes all interpolations to be processed simultaneously in the background.  Sequential processing lessens the load on the computer.

    Error Log File: This text field allows the user to specify a name for the error log file.  Since the interpolations are performed in the background, they cannot use windows or text to inform the user of an error condition.  Instead, they write the errors to the specified log file.

    The Export to Arc Grid routine creates an ascii Arc Gridfile from a binary Data Set Manager matrix data set.  The output file contains a header and data in the prescribed fashion necessary for importation into Arc Grid.  Each layer in the original matrix data set becomes a separate output file.  The names for these files are created by appending the layer number onto the root supplied in the 'Output Name' text field.

    The ascii Grid files can then be imported to Grid with the following Arc Command.

         ASCIIGRID  in_ascii_file out_grid

    Layers List:  The layers list allows the user to select any layers within the matrix which are not to be exported.

    Output Scaling: Arc Grid is unable to import a data file with values over 9999. If the 'Scale Output' toggle is selected, then all values so that the largest value in the file is less than 9999.  The scaling factor is always a multiple of ten.

    Output Name:  The name entered in this field is used as the root name for the ascii Grid files.

    The Export to Ascii Matrix routine exports the Data Set Manager's binary matrix and tabular (lonlatdata) data sets to standard ascii matrix files.  The output matrix is stored sequentially by layer.  Multiple data sets can be exported simultaneously by selecting several data sets before entering the routine.  Multiple data sets can be selected by dragging the cursor through the list.  Individual data sets can be added or removed from the current selection by holding <Ctrl> and selecting those items with the mouse.

    Header Inclusion: This option determines whether the output data file is prefixed by a header.  The header contains the data set's boundaries, the cell size, number of cells, and the cell reference used.

    Cell Reference: The cell reference option determines the coordinate reference scheme used in the header.  If 'Center' is chosen, the header coordinates refer to the center of the border cells.  If 'Lower Left Corner' is chosen, the header coordinates refer to the lower left corners of the border cells.

    Organization:  The 'by Row' option causes the data to be output in a row- major organization.  If 'by Column' is selected, the data is output in a column-major organization.  In all but the rarest cases, the 'by Row' organization should be used.

    Decimal Places Options: These options control the number of decimal places used in the output data and header.

    The Import Ascii Matrix routine converts gridded ascii data sets into the Data Set Manager format.  Input data sets must have a consistent cell size.

    Boundaries: The North, South, East, and West entry fields must be filled with the appropriate boundaries of the data set which is to be imported.  If the 'Center of Cells' (default) toggle is  selected, than the boundaries should refer to the center of the cells.  Otherwise, they should reference the lower left corner of the cells.

    Cell Size Information: The user must select to enter either the cell size or the number of cells in the data set which is to be imported. The parameter which is not provided is calculated from the given parameter and the selected boundaries.

          Dy  = Cell Size in North-South direction
          Dx  = Cell Size in East-West direction
          Ny  = Number of cells from North to South
          Nx  = Number of cells from East to West

    Data Format:  If 'by Row' option is selected, then the input data is read sequentially from the input file and the output file is filled by row from North to South (while each row is filled from  West to East).  However, if the 'by Column' option is selected, the output file is filled by column from West to East (each column is filled from North to South).  Data in the input file does not have to maintain a particular layout.  In fact, the data can even be stored sequentially since the layout is not used in determining the structure of the matrix.

    Number of Layers: This text field allows the user to specify the number of layers which are in the ascii matrix.  Layers must all have the same size and must be stored sequentially within the file.

    Missing Data Values: These textfields allow the user to enter any missing data values.  These values are then converted to -99's when the file is imported.

    File Format Option: This option selects the format of the data set which is created from the imported ascii data.  If 'DM matrix' is selected the file will be stored as a Data Set Manager matrix.  If 'DM lonlatdata' is selected, the file will be stored in a tabular Data Set Manager data set.

    Usage:  Select the files which are to be imported from the 'Files' list. Selected files must be organized as ascii matrices.  In addition, all selected matrices must have common boundaries,cell size, cell reference,  and must contain the same number of layers.

    Multiple files can be selected by dragging the cursor through the 'Files' list.  Individual files can be selected or deselected by holding <Ctrl> and then selecting files with the cursor.

    The Import Ascii Tabular routine imports tabular ascii data sets.  The data files must be in the following format.

            Longitude   Latitude   Value1 Value2 Value3  ...
            Longitude   Latitude   Value1 Value2 Value3  ...
              etc.

    Number of Data Columns: Specify the number of values contained for each longitude/latitude entry.

    Missing Data Values: These textfields allow the user to enter any missing data values.  These values are then converted to -99's when the file is imported.

    File Format Option: This option determines the type of data set that is created from the input data.  If 'DM lonlatdata' is selected, a tabular Data Set Manager data set is created.  If 'DM matrix' is selected, thedata is converted into a matrix Data Set Manager data set.  In this case, the original data must be evenly spaced (uniform cell size).

    Cell Spacing Options: If the input ascii data set has evenly spaced (gridded) data points, then the 'Uniform Cell Size' option should be selected. Missing values are permissible.

    If the input ascii data set has irregulary located data points (e.g. station data) then the 'Irregular Spacing' toggle should be selected.  In this case, the Dy/Dx textfields, and cell reference options do not apply, and the data can not be automatically converted to a matrix data set.

    Cell Size: If the data set has a uniform cell size it must be specified in the Dy and Dx entry boxes.  Dy represents the cell size in the North/South direction while Dx represents the cell size in the East/West direction.  These options are unused if the 'Irregular Spacing' option is selected.

    Cell Reference: The 'Center' option should be chosen if the longitude/latitude tags reference the center of the data cells.  The 'Lower Left Corner' option should be selected if the longitude/latitude  values refer to the lower left corners of the data cells.  For irregular data this option is forced to the 'Center' option.

    Usage:  Select the files which are to be imported from the 'Files' list. Selected files must be in the following ascii tabular format.

       Longitude   Latitude   Value1   Value2   ...

    In addition, if multiple files are selected, they must all have the same number of data columns.

    Multiple files can be selected by dragging the cursor through the 'Files' list.  Individual files can be selected or deselected by holding <Ctrl> and then selecting files with the cursor.

    The Matrix to Point Sampling routine creates point data sets by sampling matrix data sets.  A point
    data set is created for each matrix data set.  Each point data set contains the points specified in the point coordinate data set.  The data values for each point are set to the data value within the respective matrix for that geographic location.  Notice that sampling an N layer matrix will result in a point data set with N data columns.

    Details  The points in each data set created will be in the same order as the points in the point coordinate file.  Any data point which lies outside the matrix geographic extent will be placed in the point data set, but data values for all layers at that point will be set to the missing data value (-9999.0). Data points which lie on the boundary between cells are handled as follows.

    A data point on a boundary between cells in the east/west direction is given the value of the cell to the east of the boundary.

    A data point on a boundary between cells in the north/south direction is given the value of the cell to the south of the boundary.

    However, this behavior is not strictly guaranteed due to floating point imprecision.

    The Merge Missing Values routine combines the missing values in two data sets.  It is assumed that missing data values are consistent through all layers of a multi-layer data set.  For this reason, only the first layer is used to determine where the missing data values are.  Both data sets must have the same boundaries.

    Add Missing Values Option: If selected, this option merges the missing data points from  the other data set.

    File Creation Options: If 'Modify Data Set' is selected then changes are made to the original data set.  If 'Create a new Data Set' is chosen, then the revised data is saved in a new data set and the original is left unchanged.

    The Spheremap Interpolation algorithm is used to create a matrix data set from an Irregular Lonlatdata data set.  This interpolation weights neighboring data points by their respective distances.  The routine can also be used to  interpolate regular Lonlatdata data sets, when a more  stringent interpolation than Bilinear is required.

    Boundaries:  The North, South, East, and West entry boxes are used to indicate the boundaries of the region which is to be interpolated.  These boundaries default to the area of the original data set.

    Cell Size: The 'Dy' and 'Dx' entry boxes allow the user to specify the latitudinal and longitudinal size of the interpolated cells, respectively.

    Nearest Neighbors Slide: This parameter controls the number of surrounding data point from the original data set that are used to form each interpolated value.

    Distance Formula: This toggle allows the user to select the means by which distances are calculated by the algorithm.  Since the individual weighting of the data points is based on their distance fromthe point to interpolate, this option can have a considerable effect.

    Cell Reference: The 'Cell Reference' options allow the user to indicate the referencing used to indicated the boundaries of the interpolation region.  If 'Center' is chosen, then the boundaries are assumed to reference the center of the data cells.  If 'Lower Left Corner' is selected, then the boundaries are assumed to reference the lower left corner of the data cells.


    The Statistics by Region routine provides statistics (mean, min, max, standard deviation, and number of cells) by region for a data set.

    Region Data Set: The region data set defines the regions over which statistics are to be computed.  Regions are specified by an integer number in each cell belonging to a particular region.  That is, all cells containing the integer, say, 5 are considered to belong to region 5.  The region data set can be specified using the 'Set' button.  The region data set can be any matrix DM data set.  The layer scale is used to select which layer within the region data set is to be used for obtaining the regions.

    Handling of Missing Data Values:  Any cell which has a missing data value (-9999.0) in the selected layer of the region data set or has a missing data value in any layer of the data set for which statistics are being computed is considered to be a missing data cell.  No statistics (for any layer) are computed for missing data cells determined by the above criteria.  Therefore, some data might not be used if the missing data values are not consistent throughout all layers in the data set.  If this is the case, all data can be used by splitting the data set into separate data sets by layer using the Layer  Manipulation routines.  Then statistics can be computed for each layer individually.
